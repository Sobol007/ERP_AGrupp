Запись регистра сведений "Очередь системных сообщений" предназначен для хранения данных системного сообщения, поступившего в систему от адаптера ESB, содержащего в себе системную информацию.

Для каждого сообщения указывается:

  • Идентификатор сообщения - уникальный идентификатор системного сообщения, поступившего в систему из ESB (тип - Уникальный идентификатор);
  • Хранилище - сами данные, передаваемые в системном сообщении (тип - Хранилище значения);
  • Дата регистрации - дата, когда системное сообщение было зарегистрировано в очереди (тип - Дата);
  • Формат сообщения - формат, в котором записано системное сообщение (тип - перечисление "Форматы сообщений": XML, JSON, FastInfoset);
  • Класс - класс, присвоенный системному сообщению ESB (тип - Строка);
  • Дата изменения - дата, когда системное сообщение было изменено (тип - Дата);
  • Статус - текущий статус исходящего сообщения (тип - перечисление "Статусы соощений"). Может принимать значения:

    Новое - сообщение только что поступило в систему;

    Ожидание обработки - сообщение ожидает обработки;

    В обработке - сообщение находится в обработке;

    Отсутствует обработчик - для данного класса сообщения не найден обработчик;

    Обработано - сообщение обработано;

    Ошибка обработки - при выполнении обработчика возникла ошибка;

    Обработка отменена - обработка сообщения отменена;

    Ожидание отправки - сообщение ожидает отправки (для исходящих сообщений);

    Отправлено - сообщение отправлено (для исходящих сообщений);

    Отправка отменена - отправка сообщения отменена (для исходящих сообщений).
